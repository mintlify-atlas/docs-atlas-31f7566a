---
title: "Coaster type mapping"
description: "How UC2 internal coaster type identifiers map to their UC3 equivalents during conversion."
---

Ultimate Coaster 2 and Ultimate Coaster 3 use different internal identifiers for coaster types. When you convert a `.uc2sav` file, UC3Tools reads the coaster's `type` field and looks it up in a built-in mapping table to find the equivalent UC3 type. The mapped UC3 type determines which track geometry, car model, and physics profile the coaster uses in UC3.

<Warning>
If a coaster's UC2 type is not present in the mapping table, conversion will throw a `KeyNotFoundException` and the output file will not be created. You must either rename the coaster type in the `.uc2sav` file to a supported value, or request support for the missing type.
</Warning>

## How to find your UC2 coaster type

The coaster type is stored as a plain-text string in the `.uc2sav` file under the coaster's `type` field. Open the file in any text editor and search for `type=` within the coaster block. The value is case-sensitive and must match one of the UC2 Type values in the table below exactly.

| `arrowhyper` | `ar_corkscrew` |
